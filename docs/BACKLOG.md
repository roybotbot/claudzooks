# Claudzooks Backlog

## Lesson Content Improvements

- [ ] **pwd lesson: explain paths first** — Before teaching `pwd`, explain what a path is. Show an ASCII diagram of how a path is just nested sub-folders/directories.

- [ ] **Clearer command output** — Separate command output from the command prompt with a divider so it's obvious what the result looks like.

- [ ] **"Press return key" wording** — Change prompts to say "Type this command and press the return key" when asking them to try a command.

- [ ] **Don't say "this does"** — Instead of "This prints the full path..." say something like "`pwd` is a command that prints the full path..."

- [ ] **Encourage independent exploration** — Mention somewhere that they should feel free to open terminal on their own and try the commands they just learned.

- [ ] **Color/bold command names in text** — When a command like `pwd` or `ls` is referenced in instruction text, it should be a different color or bolded so it stands out.

- [ ] **Explain root in absolute vs relative paths** — In the Lesson 2 section on absolute vs relative paths, explain what "root" (`/`) actually means.

- [ ] **Don't assume knowledge** — Any references to Unix concepts should be explained. Don't assume the user knows anything.

- [ ] **Explain terminal is keyboard-only** — Mention in the beginning that terminal is different because it's all keyboard, no mouse. It was built before mice existed and is the fundamental way to interact with computers.

- [ ] **Don't jump after exercise completion** — Currently completing an exercise jumps up so you can't see the result. Instead, show a divider and then ask "Continue? (y/n)" so they can review what happened.

- [ ] **Explain trailing `/` on directories** — In the moving exercise, explain why there's a `/` at the end of a directory name (e.g., `subfolder/`).

- [ ] **Defer rm -r, add warnings** — Only show `rm` exercise initially, not `rm -r` — it's too dangerous this early. Maybe introduce `rm -r` after a test/quiz section. Or at minimum, add huge warning messages to any `rm` and `rm -r` lessons.

- [ ] **Suggest checking Finder** — When creating directories and files, suggest the user go into Finder and look to see the folder/file is actually there. Praise them for creating something in the terminal.

- [ ] **Shorten "create 10 lines" exercise** — Have the learner type the first few `echo` lines, then do the rest automatically because "they get the picture."

- [ ] **Explain head flag properly** — "You can show fewer lines with a flag" needs to explain how the `-3` flag works — it specifies the number of lines to show.

- [ ] **Explain nano is keyboard-only** — Mention explicitly that nano uses keyboard commands and arrow keys only, no mouse.

- [ ] **Separate instruction file for Claude lessons** — For lessons 6-7, the instructions need to be in a separate text file the learner can reference, because when Claude runs it pushes up previous terminal instructions and they can't scroll back to see them.
